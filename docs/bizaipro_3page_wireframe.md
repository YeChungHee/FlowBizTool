# BizAiPro 3페이지 연동 와이어프레임

## 목적
이 구조의 목적은 `학습평가 > 제안서 생성 > 이메일 컨텐츠 생성`이 각각 따로 노는 화면이 아니라, 하나의 공통 데이터 묶음을 공유하면서 자연스럽게 이어지는 3단 흐름을 만드는 것이다.

핵심 트리거는 아래 3개다.
1. `평가 결과 리포트`
2. `제안서 생성하기`
3. `이메일 컨텐츠 생성하기`

---

## 전체 흐름

```text
학습평가 페이지
  -> 평가 결과 리포트 보기
  -> 제안서 생성하기
       -> 이메일 컨텐츠 생성하기
```

공통 데이터:
- 신청업체명
- 사업자번호
- 대표자명
- 평가 점수와 등급
- 가중치 변화
- 버전 정보
- 예상 한도액
- 예상 마진율
- 예상 마진액
- 연간 기대 효과
- 주요 리스크
- 권장 다음 행동

---

## 페이지 1. 학습평가 페이지

### 페이지 목표
1. 학습 평가 결과를 본다.
2. 버전별 평가 변화와 가중치 변화를 본다.
3. `평가 결과 리포트`와 `제안서 생성하기`를 트리거한다.

### 상단 구조
- 좌측:
  - 페이지 제목: `BizAiPro 평가엔진 학습하기`
  - 현재 선택한 신청업체명
- 우측:
  - 현재 엔진 버전
  - 이번 주 학습 건수
  - 업데이트 생성 여부

### 본문 구조

#### A. 좌측 입력 패널
- 기본 자료 넣기
  - 플로우스코어 리포트
  - 상담보고서
- 추가 정보
  - 파일/이미지 업로드
  - 링크 주소 입력
  - 메모 입력
- 내부심사리포트 링크
- 학습 저장 옵션

#### B. 중앙 결과 패널
- 상태 배지
  - 거래 가능성 높음
  - 추가 확인 필요
  - 제안 비추천
- 핵심 수치 카드
  - 예상 한도액
  - 예상 마진율
  - 예상 마진액
  - 결제기간
- 기업과 거래 해석
  - 신청업체 점수 / 등급 / 해석
  - 매출처 점수 / 등급 / 해석
  - 거래구조 점수 / 등급 / 해석
  - 통합 점수 / 등급 / 해석
- 연간 기대 효과
  - `(예상 한도액 + 마진액) x 연간 가능 횟수`
  - `연간 매출 증가액`
  - `연간 영업이익 증가액`
- 먼저 확인할 점
  - 추가 확인 항목
  - 리스크 요약
- 맥킨지 스타일 미리보기
  - Executive Summary
  - Client Situation
  - Opportunity Assessment
  - Estimated Impact
  - Risks and Checks
  - Next Actions

#### C. 우측 학습/버전 패널
- 이번 건 학습 상태
- 적격 여부
- 학습 가중치
- 업데이트 생성 여부
- 현재 버전과 이전 버전 비교
- 이번 업데이트에 반영된 가중치
- 기억할 계산 기준

### 하단 Appendix
- 점수 계산 방식
- 점수와 등급 기준
- 전후 비교 리포트 링크

### 핵심 버튼
- `평가 결과 리포트`
- `제안서 생성하기`

---

## 페이지 2. 제안서 생성하기

### 페이지 목표
1. 학습평가 결과를 바탕으로 신청업체용 제안서를 만든다.
2. 맥킨지 보고서 스타일로 짧고 비즈니스톤의 문장을 제공한다.
3. `복사`, `PDF 출력`, `이메일 컨텐츠 생성하기`를 제공한다.

### 자동 입력 항목
- 신청업체명
- 사업자번호
- 대표자명

### 상단 구조
- 좌측:
  - 페이지 제목: `제안서 생성하기`
  - 신청업체명 / 대표자명 / 사업자번호
- 우측 액션:
  - `복사` 아이콘
  - `PDF 출력`

### 본문 구조

#### A. 상단 요약 바
- 제안 대상 업체
- 예상 한도액
- 예상 마진율
- 예상 마진액
- 연간 기대 효과

#### B. 보고서 본문
- Executive Summary
- Client Situation
- Opportunity Assessment
- Estimated Terms
- Expected Impact
- Risks and Checks
- Next Actions

### 문장 스타일
- 짧은 문장
- 비즈니스톤
- 결론 먼저
- 숫자보다 의미 먼저

### 하단 버튼
- `이메일 컨텐츠 생성하기`

---

## 페이지 3. 이메일 컨텐츠 생성하기

### 페이지 목표
1. 제안서 핵심 내용을 이메일 문안으로 자동 압축한다.
2. 제목 선택과 담당자 선택에 따라 문안을 자동으로 바꾼다.
3. 복사 가능한 최종 메일 본문을 제공한다.

### 자동 입력 항목
- 신청업체명
- 사업자번호
- 대표자명

### 선택 입력 항목

#### 영업담당자 선택
- `영업이사 예충희 (010-4815-1411)`
- `전략이사 박진용 (010-9600-1630)`

#### 이메일 제목 선택
- 첫 제안 드립니다
- 검토 요청드립니다
- 조건부 제안드립니다
- 추가 자료 요청드립니다
- 미팅 제안드립니다

### 본문 구조

#### A. 상단 설정 영역
- 신청업체명
- 사업자번호
- 대표자명
- 영업담당자 드롭다운
- 이메일 제목 드롭다운

#### B. 자동 생성 이메일 본문
- 시작 문구
  - `신청업체명 + 대표자명님 귀중,`
- 제목에 따른 서두
- 평가 결과 간략 요약
- 예상 한도액
- 예상 마진율
- 예상 마진액
- 연간 기대 효과
- 다음 행동 제안
- 담당자 서명

#### C. 상단 액션
- `복사` 아이콘

---

## 레이아웃 원칙

### 공통
- Apple HIG 기반
- `Apple SD Gothic Neo`
- 한 화면에 한 가지 목적
- 큰 제목, 짧은 설명
- 넉넉한 여백

### 정보 구조 원칙
1. 먼저 결론
2. 다음 숫자
3. 다음 의미
4. 마지막에 계산식

### UI 연동 원칙
1. 페이지 1 결과가 페이지 2 제안서로 이어진다.
2. 페이지 2 제안서 핵심 내용이 페이지 3 이메일로 이어진다.
3. 세 페이지는 하나의 공통 데이터 객체를 공유한다.

---

## 구현 우선순위
1. 학습평가 페이지를 허브로 확정
2. 공통 데이터 스키마 정의
3. 제안서 생성 페이지 구현
4. 복사 / PDF 출력 기능 추가
5. 이메일 컨텐츠 생성 페이지 구현
6. 제목/담당자 드롭다운 템플릿 연결
7. 세 페이지 상태 연동
