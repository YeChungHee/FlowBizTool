# 플로우페이 API 보강 규칙

플로우페이 심사엔진은 기존 수기 점수만으로 끝나지 않고, 공식 외부데이터 2가지를 추가로 반영합니다.

## 1. 한국은행 ECOS

ECOS는 `신청사/매출처 외부신호`와 `거래구조의 업황전망`을 보강합니다.

현재 반영 지표:

- 한국은행 기준금리 `722Y001 / 0101000`
- 예금은행 중소기업대출금리 `121Y006 / BECBLA0202`
- 소비자물가지수 총지수 `901Y009 / 0`
- 업종별 업황전망 BSI `512Y008 / BA / 업종코드`

점수화 방식:

- BSI가 높을수록 가점
- 중소기업대출금리, 기준금리, 물가상승률이 안정적일수록 가점
- 최종 ECOS 점수는 `0~100`으로 환산

적용 방식:

- `applicant.scores.external.ecos_signal`
- `buyer.scores.external.buyer_ecos_signal`
- `transaction.scores.macro.industry_outlook`

수기 점수와 ECOS 점수는 기본적으로 `50:50`으로 혼합합니다.

## 2. DART

DART는 공시 존재 여부와 최근 재무공시를 바탕으로 `신청사/매출처 외부신호`를 보강합니다.

현재 반영 지표:

- 법인코드 매칭 여부
- 상장/공시 법인 구분
- 설립연혁
- 최근 사업연도 매출액
- 영업이익
- 당기순이익
- 부채비율

점수화 방식:

- 공시법인으로 확인되고 최근 재무공시가 양호할수록 가점
- 부채비율이 높거나 적자가 지속되면 가점을 제한
- 최종 DART 점수는 `0~100`으로 환산

적용 방식:

- `applicant.scores.external.dart_signal`
- `buyer.scores.external.buyer_dart_signal`

수기 점수와 DART 점수는 기본적으로 `50:50`으로 혼합합니다.

## 3. 실무 해석

- 상장사나 공시대상 매출처는 DART 보강 효과가 큽니다.
- 비상장 중소기업 신청사는 DART 미매칭 가능성이 높으므로, 이 경우 기존 수기 점수를 유지합니다.
- ECOS는 업종 전망과 금리 환경을 자동 반영하므로, 업황이 빠르게 흔들릴 때 유용합니다.

## 4. 입력 예시

```json
{
  "api_enrichment": {
    "enabled": true,
    "applicant": {
      "dart_corp_name": "에이튜드",
      "ecos_industry_code": "C1400"
    },
    "buyer": {
      "dart_corp_name": "삼성전자",
      "ecos_industry_code": "C0000"
    }
  }
}
```

## 5. 한계

- ECOS는 거시/업황 보정용이라 개별 기업 부도위험을 직접 설명하지는 못합니다.
- DART는 비상장·미공시 기업에는 한계가 있어, 중소기업 실사자료와 함께 써야 합니다.
- 따라서 이 기능은 `대체`가 아니라 `보강` 레이어로 설계되어 있습니다.
