# FlowPay Web Wireframes

## 목적
이 문서는 Figma에서 바로 화면 시안으로 옮길 수 있도록 만든 웹 화면 명세서다.

디자인 키워드:
- Calm
- Clear
- Trusted
- Guided
- Apple-like

## 공통 레이아웃

### 데스크톱 기준
- 좌측 사이드바: 240px
- 상단 헤더: 72px
- 본문 최대 폭: 1440px
- 메인 콘텐츠 패딩: 32px
- 카드 라운드: 20px
- 카드 내부 패딩: 24px
- 주요 버튼 높이: 48px

### 타이포
- 페이지 제목: 32px / Semibold
- 섹션 제목: 22px / Semibold
- 카드 제목: 18px / Semibold
- 본문: 15px / Regular
- 도움말: 13px / Regular

권장 폰트:
- Apple SD Gothic Neo
- 숫자와 인터페이스 보조는 SF Pro 계열 느낌 유지

## 화면 1. 홈 대시보드

### 목적
오늘 해야 할 일을 바로 이해시키는 시작 화면

### 구조
1. 헤더
- 좌측: `FlowPay 세일즈센터`
- 우측: 사용자 프로필, 알림, 검색

2. 첫 줄 KPI 카드 4개
- `오늘 신규`
- `추가 확인`
- `제안 준비`
- `보완 요청`

3. 둘째 줄 2열 레이아웃
- 왼쪽 큰 카드: `최근 거래 검토`
- 오른쪽 카드 2개 세로 배치:
  - `최근 제안서`
  - `최근 이메일`

4. 하단 빠른 시작 영역
- 큰 버튼 3개
  - `새 거래 검토`
  - `제안서 만들기`
  - `자료 검색`

### 핵심 UX 포인트
- 숫자보다 행동 버튼이 먼저 보여야 함
- “무엇을 해야 하는지”가 바로 보여야 함

## 화면 2. 새 거래 검토 시작 - 단계형 위저드

### 목적
복잡한 영업 참고용 거래 평가 입력을 단계별로 쉽게 처리

### 상단
- 큰 제목: `새 거래 검토 시작`
- 보조문구: `아래 순서대로 입력하면 거래 가능성과 제안 조건을 자동으로 정리해드려요.`

### 좌측
- 세로 스텝퍼
  1. 신청업체
  2. 거래 구조
  3. 결제 정보
  4. 상대 회사
  5. 공식 서류
  6. 최종 확인

### 우측 메인 폼

#### Step 1. 신청업체
- 신청업체 이름
- 대표자 이름
- 사업자번호
- 업종
- 업력
- 작년 매출액

#### Step 2. 거래 구조
- 매입처 이름
- 매출처 이름
- 품목
- 매입액
- 예상 매출액

#### Step 3. 결제 정보
- 결제기간
- 예상 결제일
- 발주서 있음/없음
- 계약서 있음/없음
- 세금계산서 있음/없음
- 납품증빙 있음/없음

#### Step 4. 상대 회사
- 매입처 기본정보
- 매출처 기본정보
- 결제 경험
- 지연 이력

#### Step 5. 공식 서류
- 국세 납세증명 업로드
- 지방세 납세증명 업로드
- 4대보험 완납증명 업로드
- 법인등기부 업로드
- 기업리포트 업로드

#### Step 6. 최종 확인
- 입력 요약
- 빠진 항목 안내
- `평가 실행` 버튼

### 하단 CTA
- `이전`
- `임시 저장`
- `다음`
- 마지막 단계는 `평가 실행`

## 화면 3. 거래 가설 평가 결과

### 목적
영업 참고 결과를 숫자보다 판단과 행동 중심으로 전달

### 상단 요약 카드
- 상태 배지
  - `거래 가능성 높음`
  - `추가 확인 필요`
  - `제안 비추천`
- 업체명
- 거래명
- 실행 일시

### 핵심 수치 카드 4개
- `예상 한도액`
- `예상 마진율`
- `예상 마진액`
- `결제기간`

### 본문 2열

#### 왼쪽
- `왜 이런 결과가 나왔나요?`
  - 신청업체
  - 매출처
  - 거래구조
  - 공식서류 확인

- `월별 결제위험`
  - 1~5개월 위험 그래프

#### 오른쪽
- `추가 확인하면 좋아지는 항목`
- `지금 바로 할 수 있는 다음 단계`
- `추가 요청 자료`

### 하단 버튼
- `제안서 만들기`
- `이메일 만들기`
- `PDF로 저장`

## 화면 4. BizAiPro 제안서 생성

### 목적
거래 가설 평가 결과를 고객용 설명 문서로 즉시 전환

### 구조
1. 좌측 설정 패널
- 제안서 제목
- 대상 회사
- 제안 버전
- 조건 강조 여부
- 보완 요청 포함 여부

2. 중앙 편집 화면
- 업체 개요
- 거래 구조
- 제안 조건
- 기대 효과
- 요청 서류
- 유의사항

3. 우측 미리보기
- A4 페이지 느낌의 미리보기 카드

### 하단 버튼
- `초안 저장`
- `PDF 만들기`
- `이메일로 넘기기`

## 화면 5. 이메일 만들기

### 목적
세일즈 상황에 맞는 이메일을 빠르게 생성

### 상단
- 이메일 타입 선택 탭
  - 첫 제안
  - 조건부 제안
  - 보완 요청
  - 보류 안내

### 본문 2열
- 왼쪽: 제목과 본문 편집
- 오른쪽: 제안서/영업요약 참조 카드

### 기본 포함 요소
- 인사말
- 영업 참고 결과 한 줄 요약
- 예상 한도액 / 예상 마진율 / 결제기간
- 추가 제출 요청 또는 다음 행동
- 마무리 문장

## 화면 6. 자료 검색

### 목적
과거 자료를 재사용해 심사와 문서 품질 향상

### 구조
- 상단 큰 검색창
- 좌측 필터
  - 업종
  - 상태
  - 결제기간
  - 문서종류
- 우측 검색 결과
  - 상담보고서
  - 기업리포트
  - 제안서
  - 이메일
