# FlowPay Sales Automation Scope

## 목적 재정의
이 시스템은 `기업 심사 결과`를 확정하는 도구가 아니다.

이 시스템의 실제 목적은 아래와 같다.
- B2B 세일즈를 위한 `대략적인 기업평가 보고서`를 자동 생성한다.
- 그 보고서를 바탕으로 `거래 가능성`, `예상 한도`, `예상 마진율`을 추정한다.
- 그 결과를 바탕으로 `제안서`와 `이메일 컨텐츠`를 자동 생성한다.
- 결국 영업 담당자가 더 빠르게 제안하고, 더 정확하게 대화하고, 더 잘 후속조치할 수 있도록 돕는 `세일즈 업무 자동화 도구`다.

## 하지 않는 것
- 공식 신용등급을 부여하지 않는다.
- 법적 의미의 여신심사 결과를 확정하지 않는다.
- 내부 투자위원회 또는 최종 승인 절차를 대체하지 않는다.
- 실제 심사 승인 여부를 자동 확정하지 않는다.

## 하는 것
- 상담내용, 기업리포트, 외부 데이터, 기본 증빙을 바탕으로 `영업용 사전 판단`을 만든다.
- 거래구조를 이해하기 쉽게 정리한다.
- 거래 성사 가능성을 빠르게 가늠하게 해준다.
- 영업용 제안조건을 가설 형태로 제시한다.
- 고객에게 보낼 제안서와 이메일 초안을 만들어준다.

## 결과물의 성격
이 시스템이 내놓는 결과는 모두 `추정`, `가설`, `영업 참고용`이다.

예시:
- `거래 가능성 높음`
- `추가 확인 후 제안 권장`
- `현 단계 제안 비추천`
- `예상 한도액`
- `예상 마진율`
- `제안 시 유의사항`

즉, 결과는 `승인`이나 `거절`이 아니라 `세일즈 관점의 다음 행동 추천`에 가깝다.

## 출력 표현 기준

### 기존 표현
- APPROVE
- REVIEW
- REJECT
- 심사 결과
- 심사 엔진

### 권장 표현
- 거래 가능성 높음
- 추가 확인 필요
- 제안 비추천
- 영업 참고 결과
- 거래 가설 평가 엔진

## 세일즈 관점 핵심 질문
이 도구는 아래 질문에 답해야 한다.
1. 이 회사는 지금 제안할 만한가
2. 어떤 구조로 제안하면 성사 가능성이 높아질까
3. 어느 정도 한도와 마진으로 설명하는 게 자연스러울까
4. 어떤 리스크를 먼저 설명하거나 보완 요청해야 할까
5. 이 결과를 제안서와 이메일로 어떻게 바꿀까

## 기능 구조

### 1. 거래 가설 평가
- 신청업체, 매입처, 매출처, 거래구조를 빠르게 분석
- 거래 가능성 추정
- 한도액과 마진율 추정
- 주의사항 정리

### 2. 영업용 보고서 생성
- 기업 개요
- 거래 구조
- 강점
- 주의 포인트
- 추정 한도
- 추정 마진율
- 제안 포인트

### 3. 제안서 생성
- BizAiPro 형식의 고객 제안서 초안 생성
- 고객 상황에 맞는 설명 중심

### 4. 이메일 생성
- 첫 제안 메일
- 보완 요청 메일
- 후속 미팅 제안 메일
- 조건 안내 메일

### 5. 검색 재활용
- 기존 상담
- 기존 제안서
- 기존 이메일
- 유사 업종 사례

## 앞으로의 설계 기준
1. 점수는 `심사 점수`보다 `영업 참고 점수`로 설명한다.
2. 결과는 `승인/거절`보다 `제안 가능성` 중심으로 표현한다.
3. 공식 증빙 레이어는 법적 승인보다 `영업 리스크 확인` 용도로 사용한다.
4. 제안서와 이메일 생성이 평가엔진만큼 중요한 핵심 기능이다.
5. 검색엔진은 과거 세일즈 자산을 재사용하기 위한 핵심 축이다.
