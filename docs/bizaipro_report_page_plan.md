# BizAiPro Report Page Plan

## 페이지 목적
`BizAiPro리포트 생성 페이지`는 단순 입력 화면이 아니라 아래 4가지를 동시에 수행하는 중심 작업 화면이다.

1. 제출된 자료를 모아 `영업용 기업평가 리포트`를 생성한다.
2. 생성 과정에서 자료를 `학습 데이터`로 적재한다.
3. 누적 학습 데이터가 10건이 될 때마다 `BizAiPro 엔진`을 업데이트한다.
4. 엔진 업데이트 전후 결과를 비교하는 리포트를 생성한다.

즉, 이 페이지는 `입력 > 평가 > 제안서 생성 > 이메일 생성 > 학습 축적 > 버전업`이 이어지는 허브 페이지다.

## 제품 정의
- 공식 심사 결과를 확정하지 않는다.
- `영업 참고용 기업평가 리포트`를 만든다.
- `거래 가능성`, `예상 한도`, `예상 마진율`을 추정한다.
- 이 결과를 바탕으로 `맥킨지 보고서 스타일 제안서`를 만든다.
- 동시에 해당 건을 `평가엔진 학습 데이터`로 저장한다.

## 핵심 사용자
1. 세일즈 담당자
2. 내부 제안서 작성 담당자
3. 엔진 관리자

## 핵심 입력

### 1. 필수 입력
- 플로우스코어 리포트
- 상담보고서
- 추가 정보

`추가 정보`는 자유 입력을 지원해야 한다.
- 웹 주소
- PDF
- 이미지
- 노션 링크
- 텍스트 메모
- 엑셀/문서 파일

### 2. 추가 입력
- 내부심사리포트 노션 링크

## 핵심 출력
1. `BizAiPro리포트`
2. `거래 가능성`
3. `예상 한도액`
4. `예상 마진율`
5. `맥킨지 스타일 제안서`
6. `이메일 초안`
7. `점수 계산 방식`
8. `점수별 등급과 등급의 의미`
9. `학습 데이터 저장 여부`
10. `엔진 버전 반영 여부`
11. `업데이트 전후 비교 리포트`

## 학습 루프

### 규칙
- 리포트 1건 생성 시 학습 후보 1건 적재
- `리포트 + 상담보고서`가 한 세트로 인정될 때 학습 카운트 증가
- 10건 누적 시 엔진 업데이트 실행
- 버전명은 `v.1.주차수.01`

예:
- 1주차 첫 업데이트: `v.1.1.01`
- 2주차 첫 업데이트: `v.1.2.01`

### 업데이트 후 자동 작업
1. 기존 학습 데이터 재평가
2. 업데이트 전 버전 결과 저장
3. 업데이트 후 버전 결과 저장
4. 비교 리포트 생성

## 페이지 구성

### A. 상단 헤더
- 페이지 제목: `BizAiPro리포트 생성`
- 보조 문구: `자료를 넣으면 영업용 리포트, 제안서, 이메일을 만들고 학습 데이터로 저장합니다.`
- 우측 정보:
  - 현재 엔진 버전
  - 이번 주 학습 누적 수
  - 다음 업데이트까지 남은 건수

### B. 좌측 입력 패널

#### 1. 기본 자료 업로드
- 플로우스코어 리포트 업로드
- 상담보고서 업로드
- 추가 정보 입력

#### 2. 추가 정보 입력 방식
- 탭 또는 섹션 선택
  - 링크
  - 파일
  - 텍스트 메모

#### 3. 내부심사리포트
- 노션 링크 입력
- 링크 유효성 표시

#### 4. 학습 반영 옵션
- `이 건을 학습 데이터로 저장`
- `학습 데이터로 저장만 하고 제안서 생성은 나중에`
- `업데이트 후보에 포함`

### C. 중앙 메인 결과 패널

#### 1. 영업용 평가 결과
- 거래 가능성
- 예상 한도액
- 예상 마진율
- 예상 마진액
- 주요 포인트 3개
- 주의할 점 3개

#### 1-1. 기업과 거래 해석
- 신청업체 참고등급과 해석
- 매출처 참고등급과 해석
- 거래구조 참고등급과 해석
- 통합 참고등급과 해석

#### 1-2. 점수 계산 방식
- 신청업체 점수 계산식 그대로 표시
- 매출처 점수 계산식 그대로 표시
- 거래구조 점수 계산식 그대로 표시
- 통합 점수 계산식 그대로 표시

#### 1-3. 점수와 등급 기준
- 모든 점수는 100점 만점 기준 문구
- `A / B+ / B / C+ / C / D` 구간
- 각 등급의 쉬운 의미 설명

#### 2. BizAiPro리포트 미리보기
- 맥킨지 스타일 보고서 포맷
- 구조:
  1. Executive Summary
  2. Client Situation
  3. Opportunity Assessment
  4. Suggested Structure
  5. Estimated Terms
  6. Risks and Checks
  7. Next Actions

#### 3. 제안서 생성 버튼
- `제안서 만들기`
- `PDF로 보기`

#### 4. 이메일 생성 버튼
- `첫 제안 메일`
- `보완 요청 메일`
- `후속 미팅 메일`

### D. 우측 학습/버전 패널

#### 1. 이번 건 학습 상태
- 학습 후보 여부
- 필수 자료 충족 여부
- 품질 점검 결과

#### 2. BizAiPro 학습 현황
- 현재 누적 학습 건수
- 이번 주 업데이트 예정 여부
- 현재 버전
- 다음 버전 예상명

#### 3. 업데이트 실행 조건
- 학습 10건 충족
- 비교 리포트 생성 가능 여부
- 업데이트 생성 여부
- 이번 업데이트에 반영된 가중치

#### 4. 최근 버전 비교
- 이전 버전
- 현재 버전
- 주요 변화 요약

## 맥킨지 스타일 리포트 방향

디자인 방향:
- 하얀 배경
- 짙은 회색 타이포
- 얇은 라인과 절제된 색상
- 큰 여백
- 큰 제목과 짧은 요약
- 카드보다 보고서형 섹션 구성이 강함

문서 톤:
- 짧고 명확한 문장
- 숫자보다 시사점 먼저
- `What / So What / Now What` 구조

5. 전후 비교 리포트 자동 생성
