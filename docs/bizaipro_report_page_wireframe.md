# BizAiPro Report Page Wireframe

## 페이지 이름
`BizAiPro리포트 생성`

## 디자인 방향
- Apple HIG 기반의 절제된 인터페이스
- 한글 폰트는 `Apple SD Gothic Neo`
- 보고서 영역은 `맥킨지 스타일`
- 초등학생도 이해 가능한 쉬운 용어

## 레이아웃

### 데스크톱 기준
- 전체 폭: 1440px
- 좌측 입력 패널: 320px
- 중앙 결과 패널: 720px
- 우측 학습/버전 패널: 320px
- 상단 헤더: 88px
- 본문 패딩: 32px
- 패널 간 간격: 24px

## 상단 헤더

### 좌측
- 타이틀: `BizAiPro리포트 생성`
- 보조 문구: `자료를 넣으면 영업용 평가 리포트와 제안서를 만들고, 동시에 학습 데이터로 저장합니다.`

### 우측 상태 카드 3개
- `현재 버전`
  - 예: `v.1.4.01`
- `이번 주 학습`
  - 예: `7 / 10`
- `다음 업데이트`
  - 예: `3건 남음`

## 좌측 패널: 입력폼

### 카드 1. 기본 자료
- 제목: `기본 자료 넣기`
- 업로드 필드 1: `플로우스코어 리포트`
- 업로드 필드 2: `상담보고서`
- 설명: `파일을 끌어다 놓거나 선택하세요`

### 카드 2. 추가 정보
- 제목: `추가 정보`
- 탭 3개
  - `링크`
  - `파일`
  - `메모`

#### 링크 탭
- 입력창 placeholder: `웹 주소나 노션 링크를 넣어주세요`
- `+ 링크 추가`

#### 파일 탭
- 자유 업로드
- 허용 문구: `PDF, DOCX, XLSX, 이미지 파일 가능`

#### 메모 탭
- 큰 텍스트 에어리어
- placeholder: `회사 상황, 미팅 메모, 꼭 넣고 싶은 내용을 자유롭게 적어주세요`

### 카드 3. 내부심사리포트
- 제목: `내부심사리포트`
- 입력창: `노션 링크`
- 상태 배지:
  - `링크 확인됨`
  - `링크 확인 필요`

### 카드 4. 학습 반영 옵션
- 체크박스: `이 건을 학습 데이터로 저장`
- 체크박스: `업데이트 후보에 포함`
- 체크박스: `제안서와 이메일도 함께 생성`

### 하단 CTA
- 주 버튼: `BizAiPro리포트 생성`
- 보조 버튼: `임시 저장`

## 중앙 패널: 결과 및 문서 미리보기

### 카드 1. 영업 참고 결과
- 상단 상태 배지
  - `거래 가능성 높음`
  - `추가 확인 필요`
  - `제안 비추천`

- 핵심 수치 4개
  - `예상 한도액`
  - `예상 마진율`
  - `예상 마진액`
  - `결제기간`

- 요약 박스
  - `이 거래는 왜 제안할 수 있는가`
  - `먼저 확인해야 할 점`

### 카드 2. 기업과 거래 해석
- 신청업체 점수 / 참고등급 / 한 줄 해석
- 매출처 점수 / 참고등급 / 한 줄 해석
- 거래구조 점수 / 참고등급 / 한 줄 해석
- 통합 점수 / 참고등급 / 한 줄 해석

### 카드 3. 점수 계산 방식
- 신청업체: `financial 51.20 x 28% ... = 62.62`
- 매출처: `financial 54.90 x 32% ... = 58.89`
- 거래구조: `structure 64.96 x 45% ... = 54.57`
- 통합: `신청업체 62.62 x 40% ... = 59.30`

### 카드 4. 점수와 등급 기준
- `모든 점수는 100점 만점 기준`
- `A / B+ / B / C+ / C / D`
- 각 등급에 대한 쉬운 설명 문장

### 카드 5. BizAiPro리포트 미리보기

보고서형 A4 느낌 박스

섹션 구조:
1. `Executive Summary`
2. `Client Situation`
3. `Opportunity Assessment`
4. `Suggested Structure`
5. `Estimated Terms`
6. `Risks and Checks`
7. `Next Actions`

### 카드 6. 문서 액션
- `제안서 만들기`
- `PDF 보기`
- `이메일 만들기`

## 우측 패널: 학습 및 버전 관리

### 카드 1. 이번 건 학습 상태
- `자료 충족도`
- `학습 후보 여부`
- `저장 가능 여부`

예시 상태:
- 플로우스코어 리포트: 완료
- 상담보고서: 완료
- 추가 정보: 부분 완료
- 내부심사리포트: 확인 필요

### 카드 2. BizAiPro 학습 현황
- 현재 버전: `v.1.4.01`
- 누적 학습 건수: `7 / 10`
- 다음 버전 예상명: `v.1.4.01 -> v.1.4.02` 또는 주차 규칙에 맞는 다음 표기

### 카드 3. 업데이트 조건
- `10건 충족 여부`
- `재평가 가능 여부`
- `비교 보고서 생성 가능 여부`
- `업데이트 생성 여부`
- `이번 업데이트에 반영된 가중치`

### 카드 4. 최근 버전 변화
- 이전 버전: `v.1.3.01`
- 현재 버전: `v.1.4.01`
- 변화 요약:
  - 한도 추정 보정
  - 마진율 보정
  - 산업 점수 보정

## 사용자 흐름
1. 사용자가 리포트와 상담보고서를 넣는다.
2. 추가 링크, 파일, 메모를 함께 넣는다.
3. 내부심사리포트 노션 링크를 추가한다.
4. `BizAiPro리포트 생성`을 누른다.
5. 중앙 패널에서 영업용 리포트와 추정 조건을 확인한다.
6. 필요하면 제안서와 이메일을 바로 만든다.
7. 동시에 해당 건은 학습 데이터로 적재된다.
8. 10건 충족 시 엔진 업데이트 및 비교 리포트가 생성된다.

## 마이크로카피 예시
- `자료를 넣으면 바로 영업용 리포트를 만들어드려요`
- `이 결과는 공식 심사 결과가 아니라 제안용 참고 결과입니다`
- `추가 자료가 들어오면 조건이 더 정확해집니다`
- `이번 건을 학습 데이터로 저장할까요?`
- `10건이 쌓이면 BizAiPro 엔진이 업데이트됩니다`

## 상태별 시각 표현
- `거래 가능성 높음`: 부드러운 녹색 배지
- `추가 확인 필요`: 오렌지 배지
- `제안 비추천`: 레드 배지
- `학습 저장 완료`: 블루 배지

## Figma 제작 순서
1. 헤더와 3단 레이아웃 프레임 작성
2. 좌측 입력 패널 컴포넌트 배치
3. 중앙 리포트 미리보기 박스 제작
4. 우측 학습 상태 카드 제작
5. 상태 배지와 KPI 카드 스타일 통일
6. 모바일 1열 버전 추가
