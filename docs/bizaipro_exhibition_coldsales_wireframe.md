# BizAiPro 전시회형 제안서/이메일 와이어프레임

## 목적
전시회 참가 기업 DB를 기반으로 `플로우스코어리포트 + 웹페이지 + 대표자 정보`를 조합해
콜드세일즈용 제안서와 이메일을 만드는 화면 구조를 정의한다.

이 흐름은 상담보고서가 없는 기업에도 먼저 제안할 수 있도록 설계한다.

---

## 입력 원칙

### 최소 입력
- 기업명
- 전시회명
- 전시회정보 제공 URL
- 참가 연도
- 산업/품목
- 홈페이지
- 담당자

### 정보 해석 원칙
- `재무정보`: 플로우스코어리포트로 확인하며, 플로우페이 서비스를 제공할 수 있는 최소 기준에 미달하는지를 가르는 `필터` 역할로 둔다.
- `비재무정보`: 홈페이지, 대표자 정보, 전시회 참가 정보로 확인하며, 제안서와 이메일의 메시지 방향을 만드는 핵심 정보로 둔다.

---

## 전체 흐름

```text
전시회 DB 선택
  -> 웹페이지 / 플로우스코어리포트 확인
  -> 콜드세일즈용 간이평가
  -> 전시회형 제안서 생성
  -> 콜드메일 생성
```

---

## 페이지 1. 전시회 기업 발굴 / 간이평가

### 목표
1. 전시회 DB에서 기업을 선택한다.
2. 홈페이지와 플로우스코어리포트 유무를 확인한다.
3. 콜드세일즈용 간이평가 결과를 만든다.
4. `제안서 생성하기` 트리거를 만든다.

### 상단
- 페이지 제목: `전시회 기업 발굴`
- 모드 배지: `콜드세일즈`
- 선택 기업 기본 정보 요약

### 좌측 입력 패널
- 전시회 DB 검색/선택
- 기업명
- 전시회명
- 전시회정보 제공 URL
- 참가 연도
- 산업/품목
- 홈페이지
- 담당자
- 플로우스코어리포트 업로드 여부

### 중앙 결과 패널
- 간이평가 상태
  - 제안 우선순위 높음
  - 추가 확인 필요
  - 우선순위 낮음
- 재무 필터 결과
  - 제공 가능
  - 보수 검토
  - 제공 비추천
- 웹페이지 요약
- 전시회 참가 정보 요약
- 재무 확인 여부
- 콜드제안 포인트

### 우측 참고 패널
- 데이터 신뢰도
- 출처 목록
- 비재무 메모
- 권장 다음 행동

### 버튼
- `전시회형 제안서 생성하기`

---

## 페이지 2. 전시회형 제안서 생성하기

### 목표
1. 전시회 참가 정보를 포함한 제안서를 만든다.
2. 맥킨지 스타일 레이아웃을 유지한다.
3. 제안서 상단에 전시회 정보가 자동으로 노출된다.
4. `복사`, `PDF 출력`, `이메일 컨텐츠 생성하기`를 제공한다.

### 자동 입력
- 기업명
- 사업자번호
- 대표자명
- 전시회명
- 전시회정보 제공 URL
- 참가 연도
- 산업/품목

### 제안서 상단 추가 표기
- `전시회 참가 정보`
- `전시회명`
- `전시회정보 제공 URL`
- `참가 연도`
- `산업/품목`

### 본문 구조
- Executive Summary
- Company Snapshot
- Exhibition Context
- Opportunity Assessment
- Estimated Terms
- Expected Impact
- Risks and Checks
- Next Actions

### 액션
- `복사`
- `PDF 출력`
- `이메일 컨텐츠 생성하기`

---

## 페이지 3. 전시회형 이메일 컨텐츠 생성하기

### 목표
1. 전시회 참가 맥락이 들어간 콜드메일을 생성한다.
2. 노션의 `BizAi_pro 이메일 내용` 페이지를 참고한 제목/본문 구조를 사용한다.
3. 제안서 핵심 내용과 전시회 참가 정보를 함께 넣는다.

### 자동 입력
- 기업명
- 사업자번호
- 대표자명
- 전시회명
- 전시회정보 제공 URL
- 참가 연도
- 산업/품목

### 수동 선택

#### 영업담당자
- 영업이사 예충희 (010-4815-1411)
- 전략이사 박진용 (010-9600-1630)

#### 이메일 제목 드롭다운
노션 기준 제목 패턴을 반영한다.
- `{{회사명}}도 ‘매출성장호르몬’이 필요합니다`
- `{{회사명}}의 매출성장 타이밍을 지키는 방법`
- `원자재 매입 부담, FlowPay로 완화할 수 있습니다`
- `생산 확대와 매출 회수 사이의 공백을 메워드립니다`
- `{{회사명}}에 맞춤 원자재 선공급 제안드립니다`
- `대출이 아닌 운영자금 대안, FlowPay를 소개드립니다`

### 이메일 본문 구조
- 시작 문구
  - `{{회사명}} 귀중, 안녕하세요 {{대표자명}}님.`
- 전시회 참가 언급
  - `2026년 {{전시회명}} 참가 정보를 확인했습니다.`
- 홈페이지/플로우스코어리포트 기반 기업 해석
- 제안 핵심
  - 평가 결과 요약
  - 예상 한도액
  - 예상 마진율
  - 예상 마진액
  - 연간 기대 효과
- 미팅 유도 문장
- 담당자 서명

### 액션
- `복사`

---

## 핵심 연동 원칙
1. 전시회 기업 발굴 화면의 선택 기업 정보가 제안서로 바로 넘어간다.
2. 제안서 상단에 전시회 정보가 별도 섹션으로 표시된다.
3. 이메일은 전시회 참가 정보 + 제안서 핵심 내용을 함께 담는다.
4. 재무는 플로우스코어리포트 기준으로 `플로우페이 제공 가능 여부를 거르는 필터`로 사용한다.
5. 비재무는 웹페이지와 대표자 정보, 전시회 정보를 기반으로 `어떻게 제안할지`를 정하는 메시지 레이어로 사용한다.
